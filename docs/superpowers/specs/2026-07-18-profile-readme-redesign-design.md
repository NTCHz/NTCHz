# GitHub Profile README Redesign — Design Spec

**Date:** 2026-07-18
**Repo:** `NTCHz/NTCHz`
**Status:** Approved by user (full scope)

## Goal

Replace the template-look profile README (typing-svg, shields, skillicons,
github-readme-stats, streak-stats) with a fully self-generated terminal-themed
design matching the portfolio brand (portfolio.shipfold.com): dark background,
orange accent `#ff4d00`, monospace type. Zero external image services.

## Page Structure (top to bottom)

1. **`assets/hero.svg`** — static, committed once.
   Simulated terminal window (title bar with traffic-light dots, host
   `ntchz@shipfold`). SMIL typing animation types out name, role line
   ("Full-Stack Developer · AI & RAG · LINE Platform"), a short `cat about.txt`
   block, and links to portfolio + resume. Animation loops or ends on a
   blinking cursor.
2. **Dynamic 2×2 SVG row** — regenerated daily by GitHub Actions:
   - `assets/stats.svg` — this-year commits, PRs, top languages, drawn in the
     terminal theme (replaces github-readme-stats).
   - `assets/activity.svg` — recent commits across public repos rendered as
     `git log --oneline` (repo name + message + relative time), ~6 lines.
   - `assets/graph.svg` — contribution heatmap in orange scale (replaces
     streak-stats).
   - `assets/status.svg` — HTTP check of portfolio.shipfold.com →
     `● all systems operational` / `● degraded` status line with response time.
3. **Featured Work** — existing 16-project markdown table, kept verbatim.
4. **Open Source / Let's Talk / Connect** — existing sections, trimmed:
   remove komarev profile-views counter, skillicons strip, all
   github-readme-stats and streak-stats embeds. Keep the Connect badge row
   (shields.io badges are fine — they are links, not stats, and shields.io is
   first-party stable).

## Implementation

- **`generate.ts`** — single Bun script in this repo.
  - Fetches: GitHub GraphQL API (`contributionsCollection` for stats +
    heatmap, recent commits via `repositories → defaultBranchRef → history`),
    and `fetch("https://portfolio.shipfold.com")` timed for status.
  - Writes the 4 dynamic SVGs to `assets/`.
  - Auth: built-in `GITHUB_TOKEN` from Actions (public data only — no PAT).
- **`theme.ts`** (or inline module section) — shared tokens: colors, font
  stack (`ui-monospace, SFMono-Regular, Menlo, monospace`), card frame
  (terminal window chrome) so all cards render as one system.
- **`.github/workflows/generate.yml`** — cron daily (`0 0 * * *`) +
  `workflow_dispatch`. Runs `bun generate.ts`, commits & pushes only if
  `git diff` is non-empty.
- **Dark-only**: every SVG paints its own dark background, so it reads fine
  on GitHub light mode too. No `<picture>` variants.
- **hero.svg** is hand-authored (static file with SMIL), not generated by the
  script.

## Error Handling

- GraphQL failure → script exits non-zero, workflow fails visibly, old SVGs
  stay in place (never commit broken/empty SVGs).
- shipfold.com unreachable → that is valid data: render the degraded state in
  `status.svg`, do not fail the run.
- Text overflow in activity lines → truncate with `…` at a fixed column width
  (SVG has no text wrapping).
- Escape XML entities in commit messages before embedding in SVG.

## Testing

- `generate.test.ts` (bun test): SVG output is well-formed XML, XML escaping
  of hostile commit messages, truncation logic, degraded-status rendering.
- Manual: open generated SVGs in browser; verify README render on GitHub in
  both light and dark UI.

## Out of Scope

- Homelab-internal metrics (only public HTTP check of shipfold.com).
- Light-theme SVG variants.
- Rewriting the Featured Work table content.
